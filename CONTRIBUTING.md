# Contributing to Progress Monitoring System

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/progress-monitoring-system.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- **Backend**: Use ES6+ syntax, async/await, follow existing patterns
- **Frontend**: Functional components with hooks, use TailwindCSS
- **Python**: Follow PEP 8, use type hints where appropriate
- **Commits**: Use clear, descriptive commit messages

## Testing

Before submitting:
- Test all new features
- Ensure no console errors
- Test on both Mac and Windows if possible
- Update documentation if needed

## Pull Request Process

1. Update README.md if needed
2. Update .env.example if adding new environment variables
3. Ensure all tests pass
4. Request review from maintainers

## Questions?

Open an issue for questions or discussions about features.
